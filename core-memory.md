### 错误处理协议

**来源：用户指令 2026-03-06**

> 每犯一次错，就写一条规则；每条规则，变成一个Skill；每个Skill，确保「Never Again」

这是一条自我改进的刚性约束：
- 犯错 → 提取规则 → 技能化 → 永不再犯
- 不是道歉，是系统性的防复发
- 每个Skill都是对同类错误的疫苗

## 进化协议（Evolution Protocol）

**来源：虾的每日进化流程 🧬 2026-03-06**

- 全面复盘上月成长轨迹
- 评估技能库变化
- 设定下月进化目标

**节奏**: 每小时自检 → 每天反思 → 每周沉淀 → 每月突破

## 任务路由协议（Task Routing Protocol）

**来源：用户指令 2026-03-06**

根据任务复杂度选择执行模式：

```
任务复杂度评估
├── 简单功能扩展 → Skills
│   └── 单点能力，可复用，轻量级
├── 一次性复杂任务 → Subagent
│   └── 隔离执行，用完即走，不污染主会话
└── 持续性业务 → Team Agent
      ├── 流程固定 → 中心化管控
      │   └── 标准SOP，统一调度
      ├── 层级审批 → 分层金字塔
      │   └── 多级审核，逐层上报
      └── 个性服务 → P2P模式
            └── 点对点匹配，灵活响应
```

**决策要点：**
- **Skills**: 工具化、原子化、可组合
- **Subagent**: 复杂任务的一次性外包
- **Team Agent**: 长期业务的多角色协作架构
